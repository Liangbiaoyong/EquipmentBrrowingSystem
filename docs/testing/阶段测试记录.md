## 三、测试规范（从实践中沉淀）

### 3.1 分层测试策略

| 层         | 测试框架                                                        | Mock策略                            | Spring上下文 | 适用场景                         |
| ---------- | --------------------------------------------------------------- | ----------------------------------- | ------------ | -------------------------------- |
| Service    | `@ExtendWith(MockitoExtension.class)`                         | `@Mock` + `when().thenReturn()` | ❌ 不需要    | 业务逻辑验证、异常路径、边界条件 |
| Controller | `@WebMvcTest` + `@AutoConfigureMockMvc(addFilters = false)` | `@MockBean`                       | ✅ 部分加载  | HTTP状态码、JSON响应体、参数校验 |
| 纯工具类   | `@ExtendWith(MockitoExtension.class)` 或裸 JUnit              | 无或极少                            | ❌ 不需要    | 算法正确性、格式转换、边界条件   |

### 3.2 命名约定

```
测试类: {被测类}Test
方法:   {方法名}_{场景}_should{期望行为}

示例:
  generateToken_shouldContainAllClaims()
  localLogin_wrongPassword_shouldThrow()
  classifyByGbName_emptyInput_shouldReturnNull()
```

### 3.3 断言风格

- **AssertJ** 优先于 JUnit 原生断言（链式调用、可读性好）
  ```java
  assertThat(loginVO.getAccessToken()).isNotNull().isNotEmpty();
  assertThatThrownBy(() -> service.method())
      .isInstanceOf(BadCredentialsException.class)
      .hasMessage("用户名或密码错误");
  ```
- JSON 响应验证使用 `jsonPath`
  ```java
  .andExpect(jsonPath("$.code").value(200))
  .andExpect(jsonPath("$.data.accessToken").value("mock-jwt-token"));
  ```

### 3.4 Mock 使用原则

| 场景            | 方式                                           | 说明                            |
| --------------- | ---------------------------------------------- | ------------------------------- |
| Service 测试    | `@Mock` 字段注入                             | MockitoExtension 自动初始化     |
| Controller 测试 | `@MockBean`                                  | Spring Boot 将 mock 注册为 bean |
| 全局默认行为    | `lenient().when(...)`                        | 避免 strict stubbing 误报       |
| 多返回值        | `.thenReturn(a).thenReturn(b)`               | 第一次返回 a，第二次返回 b      |
| 异常模拟        | `.thenThrow(...)` 或 `doThrow().when(...)` | 后者用于 void 方法              |

### 3.5 测试数据构造

- **DO**: 使用辅助方法构造数据对象，避免重复代码
  ```java
  private SysUser createCasUser() { ... }
  private String csvWithHeader(String assetNo, String name) { ... }
  ```
- **DO NOT**: 在每个测试中重复写相同的对象构造代码
- **DO**: 使用清晰的变量命名区分"已有数据"和"输入数据"
  ```java
  SysUser existingUser = createCasUser();  // 数据库已有
  SysUser casUser = createCasUser();        // 当前请求
  ```

### 3.6 CSV/导入测试的特殊技巧

```
1. CSV 必须包含表头行（importFromStream 会跳过首行）
2. 列数至少 40 列（mapColumns 最长访问索引 col[37]）
3. 千分位金额需要引号: "17,500"
4. 空资产编号行会被跳过（mapColumns 返回 null）
5. 模拟分类命中/不命中控制 autoCategoryCount / uncategorizedCount
```

## 四、关键测试场景

### 4.1 自动分类算法（CategoryServiceImplTest）

```
输入: 国标分类名
算法:
  1. 去掉前缀"其他" → 去前缀后匹配
  2. 按 priority 升序遍历规则表
  3. String.contains 关键词匹配
  4. 首条命中即返回
  5. 全未命中返回 null

测试覆盖:
  ✅ "台式计算机" → 匹配"计算机" → categoryId=1
  ✅ "其他照相机及器材" → 去前缀→匹配"照相机" → categoryId=2
  ✅ "无人机航拍器" → 高优先级 priority=50 → categoryId=7
  ✅ "舞台灯光设备" → 包含匹配"舞台" → categoryId=10
  ✅ "图书资料" → 无匹配 → null
  ✅ null/空字符串 → null
  ✅ "其他" → 去前缀后空字符串 → null
```

### 4.2 CSV 导入流程（DeviceImportServiceImplTest）

```
CSV 导入管线:
  文件 → 跳过表头 → parseCsvLine(引号转义) → mapColumns(列映射)
       → classifyByGbName(自动分类) → flushBatch(去重写入)

测试覆盖:
  ✅ 正常数据 → successCount=1, autoCategoryCount=1
  ✅ 已存在资产编号 → updateCount=1, insert 不调用
  ✅ 空资产编号 → 跳过该行, totalRows=0
  ✅ 空文件 → failCount>0
  ✅ 仅表头 → totalRows=0
  ✅ 引号内逗号 → 正确解析, successCount=1
  ✅ 千分位金额 "17,500" → 正确解析, successCount=1
  ✅ 未命中分类 → uncategorizedCount=1, categoryId=10
  ✅ Dry-Run → 解析但不写入数据库
  ✅ Dry-Run 超20行 → 只取前20条
  ✅ 不支持的格式 → 抛出异常
  ✅ 批次清除 → 返回删除数
```

## 八、最佳实践总结

### 8.1 DO

- ✅ Service 测试使用纯 Mockito，不加载 Spring 上下文（快）
- ✅ Controller 测试验证 JSON 响应体（不止 status code）
- ✅ 异常路径测试：密码错误、用户不存在、被禁用、CAS 冲突
- ✅ 边界条件测试：空值、超长值、特殊字符
- ✅ 使用 `@DisplayName` 写中文描述，测试方法名英文
- ✅ 同一方法多次调用不同返回值用 `thenReturn(a).thenReturn(b)`

### 8.2 DON'T

- ❌ 不要在 Service 测试中用 `@SpringBootTest`（除非是集成测试）
- ❌ 不要用 `Thread.sleep` 测试超时（用 0ms 过期 token）
- ❌ 不要通过反射调用私有方法（耦合实现细节，更脆弱）
- ❌ 不要在 Controller 测试中依赖真实 Security 过滤链（用 `addFilters = false`）
- ❌ 不要用中文引号 `""` 写 Java 字符串（编译器会解析为字符串边界）
