## 一、分层测试策略

| 测试类型 | 工具 | 覆盖目标 | 执行时机 |
|----------|------|----------|----------|
| 单元测试 | JUnit5 + Mockito | Service / Mapper / Utils | 每次提交前本地运行 |
| 接口测试 | Postman + Newman | REST API 正确性 | 提测前运行集合 |
| 集成测试 | Spring Boot Test + Testcontainers | 数据库、Redis、MinIO 交互 | 功能模块完成时 |
| 性能测试 | JMeter | 30 并发下响应 < 2s | 核心模块完成后 |

| 兼容性测试 | 手动 | Chrome / Edge / 国产浏览器 | 前端完成 80% 时 |

## 二、单元测试规范

### 2.1 框架与工具

- **测试框架**：JUnit 5 (Jupiter)
- **Mock 框架**：Mockito
- **断言库**：AssertJ（链式断言，可读性强）
- **覆盖率工具**：JaCoCo（目标 > 70%）

### 2.2 测试范围

| 层 | 测试内容 | 是否需要 Spring 上下文 |
|----|----------|----------------------|
| Service | 业务逻辑（库存计算、冲突检测、状态流转） | 否，纯 Mock 测试 |
| Mapper | SQL 正确性 | 是，@MybatisPlusTest |
| Utils | JWT 工具、图片压缩、日期处理 | 否 |
| Controller | 参数校验、权限控制 | 是，@WebMvcTest |

### 2.3 命名约定

```
// 测试类命名：{被测类}Test
class UserServiceTest {
    // 测试方法：{被测试方法}_{场景}_should{期望行为}
    void login_validCredentials_shouldReturnToken() { ... }
    void login_wrongPassword_shouldThrowException() { ... }
}
```

### 2.4 Controller 层测试示例

```java
@WebMvcTest(AuthController.class)
class AuthControllerTest {

    @Autowired
    private MockMvc mockMvc;

    @MockBean
    private AuthService authService;

    @Test
    void login_validRequest_shouldReturnToken() throws Exception {
        LoginRequest request = new LoginRequest("admin", "admin123");
        given(authService.login(any())).willReturn("mock-jwt-token");

        mockMvc.perform(post("/api/v1/auth/local/login")
                .contentType(MediaType.APPLICATION_JSON)
                .content(new ObjectMapper().writeValueAsString(request)))
                .andExpect(status().isOk())
                .andExpect(jsonPath("$.code").value(200))
                .andExpect(jsonPath("$.data").value("mock-jwt-token"));
    }
}
```

## 六、安全测试清单

| 测试项 | 方法 | 预期结果 |
|--------|------|----------|
| 未携带 Token 访问 | 调用任意受保护 API | 返回 401 |
| Token 过期 | 使用过期 Token | 返回 401 |
| 越权访问 | 学生 Token 访问管理员接口 | 返回 403 |
| SQL 注入 | 参数中包含 `' OR 1=1--` | 查询无异常结果 |
| XSS | 提交包含 `<script>` 的内容 | 内容被转义显示 |
| 文件上传 | 上传 `.exe`、`.html` 文件 | 校验 MIME 类型，拒绝 |
| 路径遍历 | 文件名包含 `../` | 生成随机文件名，拒绝遍历 |
| 密码安全 | 检查数据库中密码存储 | BCrypt 加密，非明文 |

## 七、验收测试用例（关键业务场景）

### 场景 1：学生借用设备全流程

```
前置条件：学生已通过 CAS 登录，设备 A 可用库存 = 1
测试步骤：
1. 学生浏览设备列表，搜索设备 A
2. 点击设备 A 进入详情，选择借用时段（明天 9:00 - 17:00）
3. 填写借用事由「课程实验用」，选择导师为审批人
4. 提交申请
验证：
  - 申请单状态 = PENDING_APPROVAL
  - 设备 A 可用库存 = 1（仅预扣，未正式扣减）
  - 审批人收到待办通知

5. 使用导师账号登录，进入待审批列表
6. 查看申请详情，点击「通过」
验证：
  - 申请单状态 = PENDING_APPROVAL（流转到实验室管理员）
  - 审批记录生成

7. 使用实验室管理员账号登录
8. 审核通过
验证：
  - 申请单状态 = APPROVED
  - 设备 A 可用库存 = 0（正式扣减）

9. 学生确认借用，上传设备照片
验证：
  - 申请单状态 = BORROWING

10. 归还设备，上传归还照片
验证：
  - 申请单状态 = RETURNED
  - 设备 A 可用库存 = 1
```

### 场景 2：时间段冲突检测

```
前置条件：设备 B 可用库存 = 1，已有借用记录覆盖 2026-07-10 09:00 ~ 17:00
测试步骤：
1. 学生提交借用申请，时段为 2026-07-10 09:00 ~ 17:00
验证：
  - 后端返回冲突提示：「该时段设备不可用」
  - 借用单未创建

2. 学生提交借用申请，时段为 2026-07-10 17:00 ~ 20:00
验证：
  - 借用单创建成功（无冲突）
```

### 场景 3：权限隔离

```
测试步骤：
1. 学生 Token 访问 GET /api/v1/admin/users
验证：返回 403

2. 学生 Token 访问 POST /api/v1/devices（新增设备）
验证：返回 403

3. 学生审批不属于自己的待办
验证：返回 403
```

### 场景 4：图片压缩

```
前置条件：准备一张 10MB 的 JPG 照片
测试步骤：
1. 管理员上传设备图片
2. 借用人上传借用确认照片
3. 归还时上传归还照片
验证：
  - MinIO 中存储的文件大小 < 5MB
  - 图片宽高 ≤ 1920px
```

### 场景 5：定时清理过期图片

```
前置条件：attachment 表中存在 expire_time < now() 的记录
测试步骤：
1. 修改系统时间或手动调用清理方法
验证：
  - MinIO 中过期文件被删除
  - 数据库中对应记录的 file_url 被置空或删除
```

## 八、持续集成流水线

### 8.1 GitHub Actions 配置

```yaml
name: CI Pipeline

on:
  push:
    branches: [dev, main]
  pull_request:
    branches: [main]

jobs:
  build-and-test:
    runs-on: ubuntu-latest

    services:
      mysql:
        image: mysql:8.0
        env:
          MYSQL_ROOT_PASSWORD: root123
          MYSQL_DATABASE: device_borrow
        ports:
          - 3306:3306
      redis:
        image: redis:7-alpine
        ports:
          - 6379:6379

    steps:
      - uses: actions/checkout@v3

      - name: Set up JDK 17
        uses: actions/setup-java@v3
        with:
          java-version: 17
          distribution: 'temurin'

      - name: Backend Unit Tests
        run: cd backend && mvn clean test

      - name: Set up Node 18
        uses: actions/setup-node@v3
        with:
          node-version: 18

      - name: Frontend Build
        run: cd frontend && npm ci && npm run build

      - name: API Tests (Newman)
        run: |
          npm install -g newman
          newman run docs/testing/postman/集合.json \
            -e docs/testing/postman/环境.json
```
