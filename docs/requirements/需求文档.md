# 建筑学院设备借用系统 — 需求文档

| 版本 | 日期 | 修订内容 | 作者 |
|------|------|----------|------|
| V1.0 | 2026-07-05 | 初始完整版 | 开发团队 |

---

## 1. 项目背景

广州大学建筑学院存在大量教学科研设备（测量仪器、模型工具、摄影器材等），长期依靠纸质登记或零散电子表格管理，存在借用效率低、设备状态不透明、历史记录难追溯等问题。为规范管理、提高设备利用率、降低人为失误，开发一套集设备发布、在线预约、多级审批、归还核验、数据统计于一体的闲置设备借用系统。

## 2. 项目目标

- 实现设备全生命周期数字化管理，覆盖录入、借用、归还、维修等环节
- 提供简洁易用的 Web 界面，支持 PC 与移动端浏览器访问
- 集成广州大学 CAS 认证，自动识别师生身份
- 通过照片留证确保设备状况可追溯
- 提供数据看板，辅助学院决策设备采购与调配

## 3. 用户角色

| 角色 | 用户来源 | 核心职责 |
|------|----------|----------|
| 学生 | CAS 登录（本科生/研究生） | 浏览设备、提交借用申请、归还设备 |
| 教师 | CAS 登录（授课教师/导师） | 审批学生申请、借用设备、查看统计 |
| 实验室管理员 | 本地创建或 CAS 指定 | 设备上架管理、审批确认、维护报修、监控库存 |
| 系统管理员 | 本地独立账户 | 用户权限管理、系统配置、审批流程定义、数据备份、日志审计 |

## 4. 功能需求

### 4.1 设备管理
- **设备录入**：实验室管理员填写设备名称、型号、分类、存放位置、总数量、配件描述，上传至少一张设备照片（压缩至 5MB 内，长期保存）
- **设备列表**：支持按分类、状态、关键词搜索，分页展示，可快速查看详情
- **状态管理**：系统根据借用单自动更新设备当前可用数量，管理员可手动标记「维修中」「待报废」
- **设备详情**：显示完整信息、当前库存、历史借用记录、设备照片轮播

### 4.2 预约借用
- **设备检索**：首页展示可借设备，提供多条件筛选（分类、可用时段、关键词）
- **时段选择**：可视化日历或时间轴选择借用起止时间，系统实时检测冲突。若所选时段内设备可用库存不足则提示用户
- **提交申请**：填写借用事由、选择审批人（默认为该生导师，可根据 CAS 返回的班级信息推荐，也可手动从教师列表选择）。支持一次申请多台设备
- **我的借用**：学生/教师查看自身申请历史、当前审批进度、借用状态

### 4.3 审批流程
- **多级审批引擎**：可配置的审批流，默认两级（初审→终审），可扩展至三级（开启最终确认）
- **节点操作**：初审人/终审人收到待办通知，查看申请详情、设备状态，通过或驳回（填写意见）
- **自动流转**：当前节点通过后自动流转至下一节点，全部通过后库存扣减；驳回则终止并回滚库存
- **审批流定义**：以 JSON 快照形式存入借用单，便于追溯

### 4.4 归还管理
- **归还登记**：借用人或管理员触发归还，上传设备归还照片（压缩至 5MB，保存半年），描述设备状况
- **逾期计算**：自动判断是否超出借用时长，逾期则标记并记录逾期天数
- **损坏报修**：发现损坏可发起报修，设备状态变为「维修中」
- **历史记录**：所有借用归还记录永久保留

### 4.5 数据统计
- **仪表盘**：设备总览（总数、借出数、可用数、维修数）、本月借用趋势
- **排行榜**：热门设备 TOP10、高频借用用户 TOP10
- **利用率分析**：按分类、时段统计设备利用率（实际借出时长/总可用时长）
- **数据导出**：支持导出 Excel

### 4.6 系统管理
- **用户管理**：查看所有用户，手动创建本地账户（如实验室管理员），编辑角色
- **角色权限**：预定义角色（学生/教师/实验室管理员/系统管理员），按钮级权限控制
- **审批流配置**：设置审批节点数目、各节点可选审批人范围、是否启用三级审批
- **日志审计**：所有关键操作（登录、借用、审批、用户修改）记录并支持查询

### 4.7 消息通知
- **站内信**：申请状态变更（提交/通过/驳回）、归还提醒、逾期警告
- **WebSocket 推送**：右上角实时通知弹窗
- **邮件/企业微信**（可选扩展）：对接学校消息平台

### 4.8 CAS 单点登录
- 接入广州大学 CAS 2.0/3.0 协议，后端作为 CAS Client
- 首次登录时根据 CAS 返回属性（学工号、姓名、班级、身份类型）自动创建或更新用户
- 管理员本地账户采用用户名+密码登录（BCrypt 加密），与 CAS 账户并存

## 5. 非功能需求

| 类别 | 指标 |
|------|------|
| 并发 | 支持 30+ 并发用户 |
| 响应 | 页面响应时间 < 2s |
| 可用性 | ≥ 99.5%，支持故障恢复 |
| 安全性 | 数据加密存储，敏感操作留痕审计 |
| 可扩展 | 模块化架构，支持后续功能扩展 |
| 兼容性 | 支持 Chrome / Firefox / Edge 及国产浏览器 |
| 易用性 | 流程引导清晰，关键操作有确认提示 |

## 6. 约束条件

- 运行环境：国产 Linux（如银河麒麟 V10，内核 5.1+）
- 部署方式：Docker + Docker Compose 容器化部署
- 认证系统：必须集成广州大学 CAS
- 图片限制：借用归还照片不超过 5MB，保存周期半年
