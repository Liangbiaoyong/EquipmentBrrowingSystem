## 1. 总体架构

### 1.1 系统架构图

```
┌──────────┐     HTTP      ┌──────────┐
│ 浏览器    │ ──────────→  │  Nginx   │
│ (Vue SPA) │              │ (反向代理) │
└──────────┘              └────┬─────┘
                               │ /api/*
                               ▼
                        ┌──────────────┐
                        │ Spring Boot  │ ──→ MySQL 8.0
                        │   Backend    │ ──→ Redis 7
                        │   :8080      │ ──→ MinIO
                        └──────────────┘
```

### 1.2 前后端分离架构
- 前端 Vue 3 SPA 由 Nginx 托管静态资源
- 所有 `/api/*` 请求反向代理至 Spring Boot 后端
- 前端使用 History 模式路由，Nginx 配置 `try_files` 处理刷新 404

## 2. 技术栈

| 层次 | 技术 | 说明 |
|------|------|------|
| 前端 | Vue 3 + Vite + Pinia + Element Plus | 响应式布局，组件丰富 |
| 后端 | Spring Boot 2.7.x + MyBatis-Plus | 成熟稳定，代码生成 |
| 认证 | Spring Security + JWT + CAS Client (Apereo) | 无状态 API 认证 |
| API 文档 | Knife4j (Swagger) | 自动生成在线接口文档 |
| 数据库 | MySQL 8.0 | 持久化存储 |
| 缓存 | Redis 7 | 热点数据、黑名单、消息队列 |
| 文件存储 | MinIO | S3 兼容对象存储 |
| 图片压缩 | Thumbnailator | 纯 Java 压缩库 |
| 定时任务 | Spring @Scheduled | 过期清理、逾期提醒 |
| 部署 | Docker Compose + Nginx | 一键部署 |

## 3. 后端分层架构

```
controller  →  参数校验、路由分发
service     →  业务逻辑编排
mapper      →  MyBatis-Plus 数据访问

dto/vo      →  数据传输 / 视图对象
```

## 4. 前端架构

```
src/
├── api/          # Axios 封装，统一错误处理，JWT 自动注入

```

## 7. 安全设计

- **认证**：CAS 获取身份后签发 JWT（含用户 ID、角色列表），有效期 4 小时
- **授权**：接口使用 `@PreAuthorize` 注解校验角色
- **防越权**：后端从 Token 提取用户 ID，确保只能操作自身数据
- **文件安全**：校验 MIME 类型为 `image/png` 或 `image/jpeg`，随机文件名防路径遍历
- **密码**：本地账户密码 BCrypt 加密
- **日志**：关键操作写审计日志，不记录密码明文

## 8. 图片处理流程

```
前端上传 → 后端接收 MultipartFile
    → MIME 校验（仅 png/jpg）
    → Thumbnailator 压缩（1920px 以内，质量 0.8，<5MB）
    → 生成随机文件名
    → 上传至 MinIO
    → 记录 attachment 表
```

- 借用归还图片：`borrow-images/{yyyy-MM}/`（保存半年，定时任务清理）
- 设备图片：`device-images/`（永久保存）
- 清理任务：每天凌晨 3 点执行

## 10. 部署架构

```
docker-compose.yml
├── mysql:8.0       → 数据卷 mysql-data
├── redis:7-alpine  → 数据卷 redis-data
├── minio:latest    → 数据卷 minio-data
├── backend         → Dockerfile 构建
└── frontend        → Nginx 托管 SPA
```

所有服务在 `dev-net` 桥接网络中通信，敏感信息通过 `.env` 文件管理。
