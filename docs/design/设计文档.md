## 1. 总体架构

### 1.1 系统架构图

```
┌──────────┐     HTTP      ┌──────────┐
│ 浏览器    │ ──────────→  │  Nginx   │
│ (Vue SPA) │              │ (反向代理) │
└──────────┘              └────┬─────┘
                               │ /api/*
                               ▼
                        ┌──────────────┐
                        │ Spring Boot  │ ──→ MySQL 8.0
                        │   Backend    │ ──→ Redis 7
                        │   :8080      │ ──→ MinIO
                        └──────────────┘
```

### 1.2 前后端分离架构
- 前端 Vue 3 SPA 由 Nginx 托管静态资源
- 所有 `/api/*` 请求反向代理至 Spring Boot 后端
- 前端使用 History 模式路由，Nginx 配置 `try_files` 处理刷新 404

## 2. 技术栈

| 层次 | 技术 | 说明 |

## 7. 批量导入策略（V2更新）

### 7.1 编码检测

CSV文件自动检测编码，解决中文Windows Excel导出GBK乱码问题：
- BOM头检测（0xEF 0xBB 0xBF）→ UTF-8-BOM
- UTF-8解码后验证表头关键词"资产编号"→ UTF-8
- 否则 → GBK回退

### 7.2 Excel格式兼容

同时支持 `.xlsx`（XSSFWorkbook）和 `.xls`（HSSFWorkbook），自动识别格式。

### 7.3 智能导入三步法

以 `asset_no`（资产编号）为主键：

1. **扫描** → 新文件中所有 `asset_no` 构建集合
2. **删除** → 旧数据中不在新集合里的记录（增量同步清理）
3. **Upsert** → 已有记录→更新业务字段保留关联数据；新记录→插入并设默认值

**保留的关联数据**（导入更新时不覆盖）：
- `borrowType`（借用类型）、`laboratoryId`（所属实验室）
- `coverImage`（封面图）、`description`（备注）
- `defaultApproverId`（默认审批人）、关联的图片列表

## 8. 安全设计

- **认证**：CAS 获取身份后签发 JWT（含用户 ID、角色列表），有效期 4 小时
- **授权**：接口使用 `@PreAuthorize` 注解校验角色
- **防越权**：后端从 Token 提取用户 ID，确保只能操作自身数据
- **文件安全**：校验 MIME 类型为 `image/png` 或 `image/jpeg`，随机文件名防路径遍历
- **密码**：本地账户密码 BCrypt 加密
- **日志**：关键操作写审计日志，不记录密码明文

## 8. 图片处理流程

```
前端上传 → 后端接收 MultipartFile
    → MIME 校验（仅 png/jpg）
    → Thumbnailator 压缩（1920px 以内，质量 0.8，<5MB）
    → 生成随机文件名
    → 上传至 MinIO
    → 记录 attachment 表
```

- 借用归还图片：`borrow-images/{yyyy-MM}/`（保存半年，定时任务清理）
- 设备图片：`device-images/`（永久保存）
- 清理任务：每天凌晨 3 点执行

## 15. V2.1 新增特性

### 15.1 统计数据全局/个人范围切换

**背景**：教师默认看到自己持有设备的统计数据，管理员默认看到全局数据，但缺乏灵活的切换能力，且范式不统一（仪表盘和统计页各有一套逻辑）。

**实现**：

| 层面 | 说明 |
|------|------|
| **后端** | `/statistics/overview`、`/trend`、`/top-devices`、`/top-users`、`/utilization` 新增 `scope` 参数（`auto`/`personal`/`global`） |
| **auto 模式** | 兼容旧逻辑 — 教师(userType=1)→个人范围，实验室管理员(2)/系统管理员(3)→全局范围，学生(0)→全局 |
| **personal 模式** | 筛选 `device.custodian = 当前用户姓名`，适用于任何角色查看自己持有的设备数据 |
| **global 模式** | 不筛选，查看全部设备数据 |
| **前端开关** | 仪表盘和统计页右上角显示「个人 | 全局」RadioButton 切换，仅教师/实验室管理员/系统管理员可见 |
| **默认值** | 教师→个人、管理员→全局；用户切换后存入 localStorage 持久化 |

**影响范围**：
- `StatisticsController.java` — 重构 `getTeacherCustodian()` → `getCustodianForScope(scope)`，所有统计端点支持 scope 参数
- `Dashboard.vue` — 移除教师专属提示条，新增 scope 切换按钮，数据加载改为 `loadAll()` 函数
- `Statistics.vue` — 移除教师专属提示条，筛选栏新增 scope 切换按钮
- `statsApi.js` — 所有查询方法新增 scope 参数传递

### 15.2 审批对话框修复

**问题**：初审/终审页面点击「通过」/「驳回」后弹出的审批意见对话框缺少提交按钮，无法完成审批操作。

**根因**：`<template #footer">` 中多余的引号字符 `"` 导致 Vue 模板解析器无法正确识别 footer 插槽，按钮无法渲染。

**修复**：
- 移除 `#footer"` 中的多余引号，改为正确的 `#footer`
- 重构对话框结构：增加 `<el-form>` 包裹、`destroy-on-close` 属性、`:close-on-click-modal="false"` 防误触
- 新增 `d.submitting` 加载状态，防止重复提交
- 支持 `Ctrl+Enter` 快捷键提交
- 驳回时必填校验提示优化为「驳回时必须填写驳回原因」

### 19.2 排序方向修复

**根本原因**：Element Plus 的 `sort-change` 事件中 `order` 参数值为 `'ascending'` / `'descending'` / `null`，但后端控制器使用 `"asc".equalsIgnoreCase(order)` 判断升序。前端直接透传 `'ascending'`，导致 `"asc".equalsIgnoreCase("ascending")` 永远为 `false`，排序始终回退到降序。

**修复**：在所有排序处理函数中增加方向转换逻辑：

```javascript
// 修复前
function onSort({prop,order}){sortBy.value=prop;sortOrder.value=order||'desc';load()}

// 修复后
function onSort({prop,order}){
  sortBy.value = order ? prop : '';
  sortOrder.value = order === 'ascending' ? 'asc' : order === 'descending' ? 'desc' : '';
  load();
}
```

关键改进：
- `'ascending'` → `'asc'`（后端识别为升序）
- `'descending'` → `'desc'`（后端识别为降序）
- `null` → `''`（清除排序，恢复默认）
- `sortBy` 在清除排序时同时置空

**涉及文件（7个）**：DeviceManage.vue、DeviceList.vue、BorrowBrowse.vue、MyBorrows.vue、OverdueManage.vue、ScrapManage.vue、AdminDataTables.vue

### 19.3 侧边栏二级菜单缩进修正
